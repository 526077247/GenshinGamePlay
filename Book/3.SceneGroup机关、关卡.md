# 机关、关卡（SceneGroup）

## 一. 设计思路简介

1. 每个关卡或机关抽象成一个 **"舞台"** 的概念。本质上，玩家的一个通关流程，就是在策划的剧本下以主人公的身份完成一场舞台演出
2. 关卡里的小怪、npc 或者机关，就抽象成演出里的 **配角演员** 或 **演出道具**
3. 关卡的各种分支情况、流程阶段等同于为演出剧本的 **章节**（Suite）
4. 判断当前章节结束后就可以进入下一章节

## 二. 配置说明

### 1. 编辑器

在本工程中通过 `Tools/Graph编辑器/关卡编辑器` 打开，界面分为顶部菜单栏、左部节点编辑器、右部详情面板，编辑好后点击菜单栏中的导出。

**编辑器代码位置：** `Editor/DesignEditor/GraphEditor/SceneGroupEditor/`

| 文件 | 说明 |
|------|------|
| `SceneGroupGraphWindow.cs` | 编辑器窗口 |
| `SceneGroupGraph.cs` | Graph 数据 |
| `SceneGroupNodeView.cs` | 节点视图 |
| `SceneGroupSuitesNode.cs` | 流程阶段节点 |
| `ValueDropDownHelper.cs` | 下拉选项辅助 |
| `Trigger/` | 触发器节点（TriggerNode / TriggerActionNode / TriggerActionNodeView） |
| `Route/` | 路径节点 |

### 2. SceneGroup

一个机关或关卡，每个配置文件对应一个 SceneGroup。

**ConfigSceneGroup / SceneGroupGraph 字段说明：**

| 字段 | 说明 |
|------|------|
| Id | 全局唯一，用于索引 |
| Position | 关卡世界坐标 |
| Rotation | 关卡世界方向 |
| Actors[] | 实体数组，关卡里的小怪、npc 或者机关 |
| Zones[] | 触发区域数组，球形或者立方体，可以用来触发实体进出事件 |
| RandSuite | 是否初始随机一个流程阶段，可用于增加不确定性 |
| InitSuite | 当 RandSuite 为 false 时生效，指定初始的流程阶段 |
| Triggers[] | 事件监听数组，可以选择监听哪些事件，用来推进关卡流程进行（Graph 编辑器中通过右键 `Create/新建阶段` 创建） |
| Suites[] | 关卡的流程阶段数组，可以指定这个流程里有哪些实体、触发区、事件监听 |
| Route[] | 寻路路径数组，某些巡逻机关会用到 |

### 3. Actors 公共字段

| 字段 | 说明 |
|------|------|
| LocalId | 该 SceneGroup 唯一，用于索引 |
| Position | 初始坐标 |
| Rotation | 初始方向 |
| IsLocal | 是否是相对关卡的坐标、方向 |
| CampId | 初始阵营，影响敌友关系 |

### 4. Triggers / SceneGroupTriggerNode

| 字段 | 说明 |
|------|------|
| LocalId | 该 SceneGroup 唯一，用于索引 |
| Actions[] | 当监听到时，执行的事情，可以进行逻辑判断 |

Graph 编辑器中通过右键 `Create/监听/事件` 创建，需要将"监听"节点连接到阶段的"监听事件"节点。

### 5. Suites / SceneGroupSuitesNode

| 字段 | 说明 |
|------|------|
| Id | 该 SceneGroup 唯一，用于索引 |
| Actors[] | 该流程阶段初始登场的实体索引 |
| Zones[] | 该流程阶段初始生成的触发区域索引 |
| Triggers[] | 该流程阶段初始监听的事件索引 |
| RandWeight | 随机权值，当该 SceneGroup 的 RandSuite 为 true 时生效，值越大越容易随机到 |

Graph 编辑器中通过右键 `Create/监听/事件` 创建，需要将"监听"节点连接到"监听事件"节点。

### 6. 配置体系详细结构

配置类位于 `Code/Module/Config/SceneGroup/`，包含完整的触发器、动作、条件体系：

| 目录 | 说明 |
|------|------|
| `ConfigSceneGroupActor/` | Actor 配置 |
| `ConfigSceneGroupZone/` | 触发区域配置（球形/立方体等形状） |
| `ConfigSceneGroupSuites.cs` | 流程阶段配置 |
| `ConfigSceneGroupTrigger/` | 触发器配置 |
| `ConfigSceneGroupAction/` | 动作配置 |
| `ConfigSceneGroupCondition/` | 条件判断配置 |
| `ConfigSceneGroupConditionGenerate/` | 条件生成器 |
| `SceneGroupEvent/` | 事件类型定义 |
| `VariableValue/` | 变量值系统 |

## 四. 工作流程

```
SceneGroupManager
  └── SceneGroup（舞台）
        ├── Actors（演员：怪物/NPC/机关）
        ├── Zones（触发区域：进出检测）
        ├── Triggers（事件监听 → Actions）
        ├── Suites（流程阶段：章节切换）
        │     ├── Suite 0: 初始登场
        │     │     ├── Actors[]
        │     │     ├── Zones[]
        │     │     └── Triggers[] → 监听事件 → 推进到 Suite 1
        │     ├── Suite 1: 第二阶段
        │     └── Suite N: ...
        └── Route[]（巡逻路径）
```
