# 剧情（Story）

## 一. 概述

剧情系统通过 Graph 编辑器可视化编排剧情流程，支持串行、并行、分支、等待、对话框、Timeline 等多种节点类型。运行时由 `StorySystem` 驱动播放。

## 二. 编辑器

在本工程中通过 `Tools/Graph编辑器/剧情编辑器` 打开，界面分为顶部菜单栏、左部节点编辑器、右部详情面板，编辑好后点击菜单栏中的导出。

**编辑器代码位置：** `Editor/DesignEditor/GraphEditor/StoryEditor/`

| 文件 | 说明 |
|------|------|
| `StoryGraphWindow.cs` | 编辑器窗口 |
| `StoryGraph.cs` | Graph 数据 |
| `StoryClipNode.cs` | 剧情片段节点 |
| `StoryClipNodeView.cs` | 片段节点视图 |
| `StoryBranchClipNode.cs` | 分支节点 |
| `StoryBranchClipItemNode.cs` | 分支选项节点 |
| `StoryBranchPort.cs` | 分支端口 |
| `StoryParallelClipNode.cs` | 并行节点 |

## 三. 字段说明

### ConfigStory

| 字段 | 说明 |
|------|------|
| Actors[] | 剧情登场角色 |
| Clips[] | 剧情流程节点 |

### ConfigStoryTimeLine

支持剧情内嵌 TimeLine，需要绑定轨道对应的 GameObject。

| 字段 | 说明 |
|------|------|
| Binding<,> | key 为轨道名，value 为剧情人物 Id |

## 四. Clip 节点类型

配置类位于 `Code/Module/Config/Story/Clip/`，支持以下节点类型：

| Clip 类型 | 说明 |
|-----------|------|
| `Serial` | 串行播放，按顺序执行子节点 |
| `Parallel` | 并行播放，同时执行多个子节点 |
| `WaitTime` | 等待指定时间 |
| `Branch` | 分支选择，根据条件选择不同路径 |
| `ChangeInputState` | 切换输入状态（如禁用/启用玩家操控） |
| `CommonDialog` | 通用对话框 |
| `TimeLine` | 内嵌 Timeline 播放 |

### 其他配置

| 目录 | 说明 |
|------|------|
| `Code/Module/Config/Story/Actor/` | 剧情角色配置 |
| `Code/Module/Config/Story/Text/` | 剧情文本配置 |

## 五. 运行时代码

| 文件 | 说明 |
|------|------|

## 六. 工作流程

```
StorySystem
  └── ConfigStory
        ├── Actors[]（登场角色绑定）
        └── Clips[]（流程节点）
              ├── Serial Clip
              │     ├── ChangeInputState（禁用操控）
              │     ├── CommonDialog（对话框）
              │     ├── TimeLine（播放演出）
              │     ├── Branch（玩家选择分支）
              │     │     ├── Branch A → ...
              │     │     └── Branch B → ...
              │     ├── Parallel（并行演出）
              │     └── WaitTime（等待）
              └── ChangeInputState（恢复操控）
```
