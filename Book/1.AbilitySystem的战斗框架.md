# AbilitySystem 的战斗框架

## 一. 概述

AbilitySystem 是参考原神 GAS（Gameplay Ability System）设计的战斗框架，核心由三层结构组成：

- **Ability（能力）**：顶层触发器，定义一个完整的行为（如技能、被动、Buff 套装）
- **Modifier（修改器）**：Ability 的子项，提供持续/瞬时的逻辑块和数值修改
- **Mixin（混入器）**：Ability 和 Modifier 共用的触发器/监听器机制，决定"何时执行什么 Action"

运行时由 `AbilitySystem`（Manager）统一驱动，挂载在实体上的 `AbilityComponent` 管理当前实体的所有 Ability 实例。

### 核心代码位置

| 文件 | 说明 |
|------|------|
| `Code/Game/System/Ability/AbilitySystem.cs` | 全局管理器，驱动 Ability 生命周期 |
| `Code/Game/Component/Ability/AbilityComponent.cs` | 实体级组件，管理 Ability 添加/移除/查询 |
| `Code/Game/Component/Ability/ActorAbility.cs` | Ability 运行时实例 |
| `Code/Game/Component/Ability/ActorModifier.cs` | Modifier 运行时实例 |
| `Code/Game/Component/Ability/Mixin/` | 所有 Mixin 实现（20 种） |
| `Code/Module/Config/Ability/` | 配置类定义 |
| `Assets/AssetsPackage/EditConfig/Abilities/` | 配置文件（JSON） |

## 二. 配置说明

### 1. FSM 状态机

在本工程中 FSM 用于控制游戏角色的行为，例如玩家、怪物等实体的移动、攻击，采用动画状态机 AnimatorController 来编辑，最后导出成 json 文件，在游戏运行时使用 FsmComponent 加载配置文件然后驱动。同时会导出一份专门用于播放动画的纯净版 AnimatorController。

编辑动画状态机 FSM 的目录在 `Assets/AssetsPackage/Unit/XXX/Edit/FsmConfig.controller`，路径格式是固定的，写好后右键该文件 `工具/状态机导出` 即可。导出 json 文件和纯净版所在位置 `Assets/AssetsPackage/Unit/FsmConfig.json` 和 `Assets/AssetsPackage/Unit/XXX/Animations/FsmConfig.controller`。最后需要在 `Excel/Unit.xlsx` 表里配置相对路径才能使用。

**支持 Timeline 联合使用。** 目录需和编辑动画状态机 FSM 的目录相同，一个 `.playable` 文件对应一个动画状态机的状态，`.playable` 文件名和动画状态机的状态名保持一致。编辑时，拖入对应动画片段，使 `.playable` 文件长度和对应动画片段长度相同，然后在上面添加自定义的 Marker（如 `ExecuteAbilitySignal`）。最终会随动画状态机一起导出到 json。

相关代码：`Code/Game/Component/Fsm/`（Fsm.cs / FsmComponent.cs / FsmState.cs / FsmState.Timeline.cs）

### 2. Ability

Ability，能力，可以把它理解为用于给策划配置各种逻辑的触发器或监听器以及数值。它分为 **ActorAbility**（以下简称 Ability）和 **ActorModifier**（以下简称 Modifier），它们的功能逻辑几乎一样，区别在于 Modifier 作为 Ability 的配置子项，只能由 Ability 来触发添加。

Ability 在 `Assets/AssetsPackage/EditConfig/Abilities` 配置。

**ConfigAbility 字段说明：**

| 字段 | 说明 |
|------|------|
| AbilityName | 全局唯一，用于索引 |
| AbilitySpecials | 在此 Ability 生效的参数配置 |
| AbilityMixins | 各种触发器或监听器，可根据需求自行扩展 |
| Modifiers | 准备用来添加的整块逻辑 |

**ConfigAbilityModifier 字段说明：**

| 字段 | 说明 |
|------|------|
| ModifierName | 该 Ability 唯一，用于索引 |
| Duration | 持续时间，-1 无限，0 瞬时，>0 有限时间，单位 ms |
| StackingType | 叠加类型，Duration 不为 0 时有效 |
| StackLimitCount | 叠加上限，StackingType 为 Multiple 时有效 |
| Mixins | 各种触发器或监听器，可根据需求自行扩展 |
| Properties | 用于修改玩家数值 |

### 3. Mixin 体系

Mixin 是 Ability/Modifier 的核心扩展机制。每种 Mixin 负责监听一种事件并在触发时执行配置的 Action 列表。

**当前已实现的 Mixin（`Code/Game/Component/Ability/Mixin/`）：**

| Mixin | 触发时机 |
|-------|---------|
| `AttachToNormalizedTimeMixin` | 动画归一化时间达到指定值 |
| `AttachToStateIDMixin` | 进入指定 FSM 状态 |
| `DoActionAfterAddMixin` | Ability/Modifier 添加后 |
| `DoActionAfterAttackMixin` | 攻击命中后 |
| `DoActionAfterBeAttackMixin` | 被攻击后 |
| `DoActionAfterLoadObjectMixin` | 对象加载完成后 |
| `DoActionBeforeAttackMixin` | 攻击前 |
| `DoActionBeforeBeAttackMixin` | 被攻击前 |
| `DoActionBeforeRemoveMixin` | Ability/Modifier 移除前 |
| `DoActionByExecuteMixin` | 手动 Execute 时 |
| `DoActionByGadgetStateMixin` | Gadget 状态变化时 |
| `DoActionByStateIDMixin` | FSM 状态匹配时 |
| `DoActionByTickMixin` | 定时 Tick |
| `DoActionOnColliderBoxMixin` | 碰撞盒事件 |
| `DoActionOnInputMixin` | 输入事件 |
| `DoActionOnInteeTouchMixin` | 交互触发 |
| `DoActionOnTriggerMixin` | 触发器事件 |
| `DoActionOnUseSkillMixin` | 使用技能时 |

### 4. Action 体系

Mixin 触发后执行配置的 Action 列表。Action 配置位于 `Code/Module/Config/Ability/Action/`，当前包含约 31 种 Action 类型，如：

- `ApplyModifier` — 添加 Modifier
- `CreateBullet` — 创建子弹
- `CreateGadget` — 创建生成物
- `TriggerAttackEvent` — 触发攻击判定
- `SetFsmParam` — 设置 FSM 参数
- `TryDoSkill` — 尝试释放技能
- `KillSelf` — 自杀

**ConfigAbilityAction 公共字段说明：**

| 字段 | 说明 |
|------|------|
| Predicate | Targetting 目标重新选定生效前判定是否满足条件，不满足则不执行 |
| Targetting | 重新选定 Action 的目标 |
| OtherTargets | Targetting 为 Other 时有效 |
| PredicateForeach | Targetting 目标重新选定生效后对每一个目标进行判定是否满足条件，不满足则不执行 |

### 5. ConfigActor

用于描述实体的基础属性，比如初始附加的 Ability、属性、武器挂点等。最后需要在 `Excel/Unit.xlsx` 表里配置相对路径才能使用。

**ConfigActor 字段说明：**

| 字段 | 说明 |
|------|------|
| Abilities | 初始附加的 Ability |
| Common | 身高、缩放等 |
| Combat | 战斗属性、死亡效果 |
| EquipController | 装备挂点 |
| Billboard | 头顶信息 |
| Intee | 交互信息 |
