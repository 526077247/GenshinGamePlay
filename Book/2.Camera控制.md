# Camera 控制

## 一. 设计思路简介

1. 这是一个基于**状态栈**的相机框架，状态有优先级，维护一个状态栈，当栈顶相机变化时过渡到新的状态
2. 每个状态包括三部分内容：
   - **Head**：更新朝向（lookAt 方向）
   - **Body**：更新坐标（位置跟随）
   - **Other**：其他效果（相机震动、物体遮挡前推等）
3. 当栈顶改变需要过渡时，同时压入一个 **Blender 状态**，用来处理 From 和 To 的过渡，过渡完了后自己弹出，刚好最顶就是新状态

## 二. 核心代码结构

### 代码位置

| 文件 | 说明 |
|------|------|
| `Code/Module/Camera/CameraManager.cs` | 相机管理器主入口 |
| `Code/Module/Camera/CameraManager.Stack.cs` | 状态栈管理 |
| `Code/Module/Camera/CameraManager.URP.cs` | URP 渲染管线适配 |
| `Code/Module/Camera/CameraShakeParam.cs` | 震动参数定义 |
| `Code/Module/Camera/CursorStateType.cs` | 鼠标光标状态 |

### 状态类

| 文件 | 说明 |
|------|------|
| `State/CameraState.cs` | 相机状态基类 |
| `State/CameraStateData.cs` | 状态数据 |
| `State/NormalCameraState.cs` | 常规相机状态（玩家操控） |
| `State/BlenderCameraState.cs` | 过渡混合状态（栈顶切换时自动压入） |

### 插件运行器（Plugin Runner）

每个状态由三类插件驱动，位于 `Code/Module/Camera/Runner/`：

**Head（朝向）：**

| 插件 | 说明 |
|------|------|
| `CameraHardLookAtPluginRunner` | 硬锁定看向目标 |
| `CameraThirdPersonLookAtPluginRunner` | 第三人称看向目标 |

**Body（位置）：**

| 插件 | 说明 |
|------|------|
| `CameraHardLockToTargetPluginRunner` | 硬锁定跟随目标 |
| `CameraThirdPersonFollowPluginRunner` | 第三人称跟随 |

**Other（其他）：**

| 插件 | 说明 |
|------|------|
| `CameraShakePluginRunner` | 相机震动 |
| `CameraColliderPluginRunner` | 遮挡前推（防止穿墙） |

### 配置类

配置位于 `Code/Module/Config/Camera/`，包含 `ConfigCamera.cs`、`ConfigCameras.cs` 及 `Plugin/` 子目录。

## 三. 工作流程

```
CameraManager（状态栈）
  ├── NormalCameraState（玩家操控相机）
  │     ├── Head: ThirdPersonLookAt
  │     ├── Body: ThirdPersonFollow
  │     └── Other: Collider / Shake
  │
  ├── [技能/剧情触发] → 压入新状态
  │     └── BlenderCameraState（自动插入，处理过渡）
  │           └── 过渡完成后自动弹出
  │
  └── 新状态成为栈顶
```

1. `CameraManager` 在 `Entry.StartGameAsync()` 中注册并 `LoadAsync()` 初始化
2. 默认压入 `NormalCameraState` 供玩家操控
3. 战斗技能、剧情等场景可压入高优先级状态，自动触发 Blender 过渡
4. 效果结束后弹出状态，栈恢复到前一个状态
