# 前置知识点

本框架基于 ET 架构思想，使用单线程异步、Entity-Component 模式。在阅读后续文档前，建议先了解以下概念。

## 框架核心概念

### Entity-Component 架构

- **Entity**：实体基类，包含唯一 Id、组件容器。Actor / Avatar / Character / Monster / Gadget / Unit / Zone / SceneGroup / Equip / Effect / SceneEntity 均派生自 Entity
- **Component**：组件，挂载在 Entity 上提供功能。Ability / AI / Combat / Fsm / Move / Numeric / SceneGroup / Billboard / Equip / Input / Intee / Pose / Trigger / View 等
- **System / Manager**：全局管理器，通过 `ManagerProvider.RegisterManager<T>()` 注册，驱动各组件的 Update 逻辑

### 管理器注册流程

入口 `Entry.Start()` → `Entry.StartAsync()` 分两阶段注册：

1. **框架层**（Mono）：Messager / Log / Attribute / CoroutineLock / Timer / Cache / Config / Resources / GameObjectPool / I18N / UI
2. **游戏层**（Code）：Performance / ImageLoader / Material / Scene / Camera / Input / Navmesh / Sound / Ability / Model / Billboard / Fsm / Move / Camp

### 单线程异步

- 使用 **ETTask** 实现单线程异步，所有异步逻辑在同一线程上轮转，无需锁
- `async ETTask` / `await` 语法，`.Coroutine()` 启动不等待
- 配合 **CoroutineLock** 协程锁防止并发冲突
- 配合 **TimerManager** 实现延时、循环定时器

### 配置系统

- Excel → 导表工具 → Nino 二进制序列化（`.bytes`）/ JSON
- 运行时通过 `ConfigManager` + `ConfigAttribute` 标记的类型自动加载
- 反序列化后调用 `ISupportInitialize.EndInit()` 完成后处理

### 资源管理

- **YooAsset** 管理所有资源加载与热更新
- 支持 EditorSimulateMode / HostPlayMode / WebPlayMode 三种模式
- `ResourcesManager` 封装异步加载，`GameObjectPoolManager` 管理对象池

### 热更新

- **HybridCLR** 实现 C# 代码热更
- `CodeLoader` 负责加载 Hotfix DLL，支持运行中重载（`ReStart()`）
- `CodeMode` 枚举：LoadDll / BuildIn / LoadFromUrl

### 代码混淆

- **Obfuz** 加密虚拟机在 `Init.SetUpStaticSecret()` 中初始化
- 从 `Resources/Obfuz/defaultStaticSecretKey` 加载密钥
- 被混淆的代码需在虚拟机初始化后才能运行
